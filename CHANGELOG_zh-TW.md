# 2.1.89 版本更新

- 新增 `"defer"` 權限決策至 `PreToolUse` hooks — 無頭模式的會話可以在工具呼叫處暫停，並使用 `-p --resume` 繼續以讓 hook 重新評估
- 新增 `CLAUDE_CODE_NO_FLICKER=1` 環境變數以選用無閃爍的 alt-screen 渲染，搭配虛擬化的捲軸回溯
- 新增 `PermissionDenied` hook，在自動模式分類器拒絕後觸發 — 返回 `{retry: true}` 以告訴模型它可以重試
- 新增具名子代理至 `@` mention 的預輸入建議
- 新增 `MCP_CONNECTION_NONBLOCKING=true` 供 `-p` 模式使用，以完全跳過 MCP 連線等待，並將 `--mcp-config` 伺服器連線限制在 5 秒而非阻塞最慢的伺服器
- 自動模式：被拒的指令現在顯示通知，並在 `/permissions` → Recent 標籤中出現，您可以使用 `r` 重試
- 修正 `Edit(//path/**)` 和 `Read(//path/**)` 允許規則以檢查已解析的符號連結目標，而非僅檢查請求的路徑
- 修正語音推送說話功能不針對某些修飾鍵組合綁定啟用，以及 Windows 上的語音模式因「WebSocket upgrade rejected with HTTP 101」而失敗
- 修正 Edit/Write 工具在 Windows 上雙倍 CRLF 並刪除 Markdown 硬換行符（兩個尾部空格）
- 修正 `StructuredOutput` schema 快取錯誤，導致使用多個 schemas 時約 50% 的失敗率
- 修正記憶體洩漏，其中大型 JSON 輸入在長時間執行的會話中作為 LRU 快取金鑰被保留
- 修正移除來自超大會話檔案（超過 50MB）中的訊息時的當機
- 修正伺服器崩潰後的 LSP 伺服器殭屍狀態 — 伺服器現在在下一次請求時重新啟動，而非失敗直到會話重新啟動
- 修正在 `~/.claude/history.jsonl` 中的 4KB 邊界處當 CJK 或 emoji 包含時提示歷史記錄項目被無聲刪除
- 修正 `/stats` 因排除子代理使用而低估令牌計數，以及當統計快取格式變更時丟失超過 30 天的歷史資料
- 修正 `-p --resume` 當延遲的工具輸入超過 64KB 或不存在延遲標記時掛起，以及 `-p --continue` 不繼續延遲工具
- 修正 `claude-cli://` 深層連結在 macOS 上無法開啟
- 修正 MCP 工具錯誤截斷至僅第一個內容區塊，當伺服器返回多元素錯誤內容時
- 修正透過 SDK 傳送含圖片訊息時被刪除的技能提醒和其他系統上下文
- 修正 PreToolUse/PostToolUse hooks 為 Write/Edit/Read 工具接收 `file_path` 作為絕對路徑，符合已記錄的行為

- 修正嵌套 CLAUDE.md 檔案在長會話中讀取許多檔案時被重複注入數十次
