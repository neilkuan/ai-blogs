Claude Code 2.1.85 更新日誌

翻譯自 官方 CHANGELOG

2.1.85

  • 新增 CLAUDE_CODE_MCP_SERVER_NAME 和 CLAUDE_CODE_MCP_SERVER_URL 環境變數至 MCP headersHelper 指令碼，允許一個幫助程式服務多個伺服器
  • 新增條件 if 欄位供 hooks 使用權限規則語法（例如 Bash(git *)），以過濾何時執行並減少行程生成開銷
  • 新增時間戳記標記至文字記錄，當排定的任務（/loop、CronCreate）觸發時
  • 新增 [Image #N] 預留位置後尾部空格（貼上影像時）
  • 深層連結查詢（claude-cli://open?q=…）現在支援最多 5,000 個字元，長預填提示時帶「scroll to review」警告
  • MCP OAuth 現在遵循 RFC 9728 Protected Resource Metadata 探索以尋找授權伺服器
  • 組織政策（managed-settings.json）阻止的外掛無法再被安裝或啟用，並在市場檢視中隱藏
  • PreToolUse hooks 現在可以藉由返回 updatedInput 與 permissionDecision: "allow" 滿足 AskUserQuestion，啟用無頭整合以透過其自身 UI 收集答案
  • OpenTelemetry tool_result 事件中的 tool_parameters 現在限於 OTEL_LOG_TOOL_DETAILS=1 後面
  • 修正 /compact 當對話成長太大以至於緊湊請求本身無法容納時失敗「context exceeded」
  • 修正 /plugin enable 和 /plugin disable 當外掛安裝位置與在設定中聲明的位置不同時失敗
  • 修正 --worktree 在非 git 儲存庫中以錯誤結束，WorktreeCreate hook 可執行之前
  • 修正 deniedMcpServers 設定不阻止 claude.ai MCP 伺服器
  • 修正 switch_display 在多監視器設定上在電腦使用工具中返回「not available in this session」
  • 修正當 OTEL_LOGS_EXPORTER、OTEL_METRICS_EXPORTER 或 OTEL_TRACES_EXPORTER 設定為 none 時當機
  • 修正非原生組建中的 diff 語法突出顯示無法運作
  • 修正 MCP 提升授權在刷新令牌存在時失敗 — 透過 403 insufficient_scope 要求提升範圍的伺服器現在正確觸發重新授權流程
  • 修正遠端會話中的記憶體洩漏，當串流回應被中斷時
  • 修正邊界連線流失期間的持續 ECONNRESET 錯誤，透過重試時使用新鮮 TCP 連線
  • 修正提示在執行某些斜杠指令後被卡在佇列中，上箭頭無法檢索
  • 修正 Python Agent SDK：type:'sdk' MCP 伺服器透過 --mcp-config 傳遞在啟動期間被丟棄
  • 修正在 SSH 上或 VS Code 整合終端中執行時在提示中洩漏的原始金鑰序列
  • 修正遠端控制會話狀態在權限解決後保持卡在「Requires Action」
  • 修正 shift+enter 和 meta+enter 被預輸入建議攔截而非插入換行符
  • 修正串流期間捲軸向上時舊內容滲漏
  • 修正終端在 Ghostty、Kitty、WezTerm 和其他支援 Kitty 鍵盤協議的終端中的退出後保留在增強鍵盤模式 — Ctrl+C 和 Ctrl+D 現在在退出後正確運作
  • 改進 @-mention 檔案自動完成在大型儲存庫上的效能
  • 改進 PowerShell 危險指令偵測
  • 改進捲軸效能，具有大型文字記錄，透過以純 TypeScript 實現替換 WASM yoga-layout
  • 減少自動壓縮在大型會話上觸發時的 UI 卡頓
此頁面由自動化流程產生，僅供參考。
