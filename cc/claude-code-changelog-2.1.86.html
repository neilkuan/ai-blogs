Claude Code 2.1.86 更新日誌

翻譯自 官方 CHANGELOG

2.1.86

  • 新增 X-Claude-Code-Session-Id 標頭至 API 請求，使代理可以按會話聚合請求而無需解析本體
  • 新增 .jj 和 .sl 至 VCS 目錄排除清單，使 Grep 和檔案自動完成不下降至 Jujutsu 或 Sapling 中繼資料
  • 修正 --resume 在會話建立於 v2.1.85 之前時失敗「tool_use ids were found without tool_result blocks」
  • 修正配置條件技能或規則時，Write/Edit/Read 在專案根目錄外的檔案上失敗（例如 ~/.claude/CLAUDE.md）
  • 修正可能在每次技能呼叫時造成效能問題和 Windows 上設定損毀的不必要設定磁碟寫入
  • 修正 /feedback 在非常長會話中包含大型文字記錄檔案時的潛在記憶體不足當機
  • 修正 --bare 模式在互動式會話中丟棄 MCP 工具並無聲丟棄中途排隊的訊息
  • 修正 c 快速鍵僅複製約 20 個 OAuth 登入 URL 字元而非完整 URL
  • 修正掩蔽輸入（例如 OAuth 代碼貼上）在狹窄終端上多行換行時洩漏令牌開始
  • 修正官方市場外掛指令在 v2.1.83 以來的 macOS/Linux 上失敗「Permission denied」
  • 修正執行多個 Claude Code 實例且在其中一個使用 /model 時狀態列顯示另一會話的模型
  • 修正長對話中滾輪滾動或點擊選取後捲軸未跟隨新訊息
  • 修正 /plugin 卸載對話：按 n 現在正確卸載外掛而保留其資料目錄
  • 修正按下 Enter 後點擊可能留下文字記錄空白直到回應到達的迴歸
  • 修正 ultrathink 提示在刪除關鍵字後逗留
  • 修正長會話中的記憶體增長，來自保留完整內容字串的 markdown/highlight 渲染快取
  • 減少當許多 claude.ai MCP 連接器配置時的啟動事件迴圈停滯（macOS keychain 快取從 5 秒延長至 30 秒）
  • 減少提及具有 @ 的檔案時的令牌開銷 — 原始字串內容不再進行 JSON 轉義
  • 透過從工具描述移除動態內容改進 Bedrock、Vertex 和 Foundry 使用者的提示快取命中率
  • 記憶體檔名在「Saved N memories」通知中現在在懸停時突出顯示，並在點擊時開啟
  • /skills 清單中的技能描述現在限制在 250 個字元以減少上下文使用
  • 變更 /skills 功能表以按字母順序排序以便更輕鬆的掃描
  • 當由計劃限制停用時，自動模式現在顯示「unavailable for your plan」（之前是「temporarily unavailable」）
  • [VSCode] 修正延長執行操作期間擴充功能不正確地顯示「Not responding」
  • [VSCode] 修正擴充功能在 OAuth 令牌刷新後將 Max 計劃使用者預設為 Sonnet（登入後 8 小時）
  • 讀取工具現在使用緊湊的行號格式並去重複未變更的重新讀取，減少令牌使用
