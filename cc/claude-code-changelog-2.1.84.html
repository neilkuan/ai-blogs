Claude Code 2.1.84 更新日誌

翻譯自 官方 CHANGELOG

2.1.84

  • 新增 PowerShell 工具作為 Windows 上的自願預覽。在 https://code.claude.com/docs/en/tools-reference#powershell-tool 深入瞭解
  • 新增 ANTHROPIC_DEFAULT_{OPUS,SONNET,HAIKU}_MODEL_SUPPORTS env vars 以覆蓋 3p 的努力/思考能力偵測（Bedrock、Vertex、Foundry），以及 _MODEL_NAME/_DESCRIPTION 以自訂 /model 選擇器標籤
  • 新增 CLAUDE_STREAM_IDLE_TIMEOUT_MS env var 以設定串流閒置監視狗臾值（預設 90 秒）
  • 新增 TaskCreated hook，當透過 TaskCreate 建立任務時觸發
  • 新增 WorktreeCreate hook 對 type: "http" 的支援 — 透過回應 JSON 中的 hookSpecificOutput.worktreePath 返回建立的 worktree 路徑
  • 新增 allowedChannelPlugins 團隊/企業管理員的受控設定以定義通道外掛允許清單
  • 新增 x-client-request-id 標頭至 API 請求以供除錯逾時
  • 新增閒置返回提示，當使用者在 75+ 分鐘後返回時輕推至 /clear，減少不必要的令牌重新快取在過時會話上
  • 深層連結（claude-cli://）現在在您偏好的終端中開啟而非恰好第一個檢測到的終端
  • 規則和技能 paths: 前言現在接受 YAML globs 清單
  • MCP 工具描述和伺服器指令現在限制在 2KB 以防止 OpenAPI 生成的伺服器膨脹上下文
  • 本地和透過 claude.ai 連接器配置的 MCP 伺服器現在被去重 — 本地設定獲勝
  • 卡在互動式提示上的背景 bash 任務現在在約 45 秒後浮出通知

  • 修正語音推送說話：保持語音鍵不再洩漏字元至文字輸入，文字記錄現在在正確位置插入
  • 修正上/下箭頭鍵在頁尾項目聚焦時無回應
  • 修正 Ctrl+U（kill-to-line-start）在多行輸入中的行邊界上為無操作，所以重複的 Ctrl+U 現在跨行清除
  • 修正空綁定預設弦綁定（例如 "ctrl+x ctrl+k": null）仍進入弦等待模式而非釋放前綴鍵
